# Study Material Generator

## Overview

将长篇教育内容转换为结构化、易学习的材料，按章节组织，每章包含学习时长、学习目标、核心内容摘要和练习题。

## Workflow

遵循以下 6 步流程生成结构化学习材料：

### Step 1: 分析源内容

阅读并理解源文档（PDF、markdown 或文本文件）：

1. **识别内容结构** - 找出自然的章节、主题或部分
2. **估算长度** - 确定合适的章节划分（每章约 15 分钟）
3. **提取核心概念** - 识别主要主题、子主题和重要细节

**📤 Expected Output:**

- 源文档的结构分析
- 建议的章节数量（3-10 章）
- 每章的主题范围说明

### Step 2: 规划章节结构

将内容划分为逻辑章节：

1. **确定章节数量** - 基于内容长度和复杂度（通常 3-10 章）
2. **设定学习时长** - 每章约 15 分钟阅读时间
3. **定义章节范围** - 每章应覆盖一个连贯的主题领域

**📤 Expected Output:**

- 章节大纲（包含标题和范围）
- 每章的学习时长
- 章节间的逻辑关系说明

### Step 3: 提取章节内容

为每章提取和组织：

1. **学习目标** - 3-5 个具体、可执行的目标
2. **核心内容** - 总结关键概念、定义和解释
3. **重要细节** - 包含示例、图表或关键信息

**📤 Expected Output:**

- 每章的学习目标列表
- 章节核心内容摘要（带子节划分）
- 关键术语和定义列表

### Step 4: 生成练习题

为每章创建多项选择题：

1. **题目数量**：每章 5-10 题
2. **难度分布**：混合基础题、中等题和进阶题
3. **覆盖范围**：确保题目涵盖所有章节主题
4. **干扰项**：为每题创建 3 个看似合理的错误选项

**📤 Expected Output:**

- 每章的练习题（带正确答案标注）
- 题目难度分布说明
- 每题对应的章节知识点

**Question Format Template:**

```markdown
1. [问题文本]
   - A) [选项 1]
   - B) [选项 2]
   - C) [选项 3]
   - D) [选项 4]
```

> 详细题目设计指南请参考：[references/question_guidelines.md](references/question_guidelines.md)

### Step 5: 组装完整文档

将所有章节编译为单个 markdown 文档：

1. **文档头部** - 标题、学习说明、概述
2. **目录** - 每章的链接
3. **章节内容** - 编号部分及所有内容
4. **答案键** - 可选：在文档末尾提供正确答案

**📤 Expected Output:**

- 完整的 markdown 文档
- 结构化的目录
- 所有章节的完整内容

> 完整文档结构模板请参考：[references/structure_template.md](references/structure_template.md)

### Step 6: 格式化和润色

1. **使用一致的格式** - 应用 markdown 标题、列表和强调
2. **添加视觉结构** - 使用 emoji、图标或章节分隔符
3. **检查完整性** - 验证所有章节都有目标、内容和问题
4. **检查可读性** - 确保语言清晰且适合目标受众

**📤 Expected Output:**

- 格式统一的最终文档
- 视觉元素（emoji、图标）
- 质量检查清单（完整性、可读性）

## 🛠️ Best Practices

### 内容提取

- **保留关键术语** - 保持重要术语和定义的完整性
- **简化复杂解释** - 用更清晰的语言重新表述过于技术性的内容
- **保持逻辑流畅** - 确保概念逐步构建

### 题目设计

- **避免琐碎问题** - 侧重理解而非单纯记忆
- **使用看似合理的干扰项** - 让错误选项具有迷惑性
- **测试不同认知层次** - 包含记忆、理解和应用类题目
- **平衡覆盖** - 题目均匀分布覆盖所有章节主题

### 格式化

- **使用中文语言** - 如果内容是中文，保持一致性
- **添加视觉提示** - 使用 emoji 使内容更吸引人（⏱️ 时间、🎯 目标、📚 内容、📝 问题）
- **保持章节一致** - 所有章节使用相同的结构

## ❓ Troubleshooting

### 问题 1: 如何确定章节划分？

**症状：** 源文档没有明确的章节，难以划分

**解决方案：**

- 寻找主题转换点（如标题、小标题）
- 基于概念复杂度划分（每章 1-3 个核心概念）
- 每章目标长度控制在 2000-3000 字

### 问题 2: 练习题难度如何把握？

**症状：** 生成的题目过于简单或过于困难

**解决方案：**

- 参考题目难度分布：基础题 40%、中等题 40%、进阶题 20%
- 基础题：测试概念记忆和定义
- 中等题：测试理解和简单应用
- 进阶题：测试综合分析和复杂应用

### 问题 3: 如何处理技术性很强的内容？

**症状：** 内容包含大量专业术语和复杂公式

**解决方案：**

- 保留核心术语，但在首次出现时给出定义
- 将复杂公式拆解为步骤说明
- 添加实际应用示例帮助理解
- 减少不必要的细节，聚焦核心概念

### 问题 4: 练习题干扰项如何设计？

**症状：** 错误选项太明显或多个正确答案

**解决方案：**

- 干扰项应该与主题相关但包含常见误解
- 避免使用"以上都对"或"以上都不对"
- 确保只有一个选项明确正确
- 让干扰项看起来合理但有关键错误

### 问题 5: 学习时长估算不准确？

**症状：** 标注 15 分钟但实际需要更长时间

**解决方案：**

- 按每分钟阅读 200-250 字估算
- 练习题预计每题 1-1.5 分钟
- 根据内容复杂度调整：简单内容可缩短，复杂内容可延长
- 在"学习说明"中说明这是估算时间
