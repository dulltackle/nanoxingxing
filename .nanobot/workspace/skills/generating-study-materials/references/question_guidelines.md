# 多项选择题设计指南

## 基本原则

### 题目数量

- 每章5-10道选择题
- 根据章节内容复杂度调整
- 确保覆盖本章所有核心概念

### 难度分布

- **基础题（40%）**：测试基本概念和定义的记忆
- **中等题（40%）**：测试理解和应用能力
- **提高题（20%）**：测试综合分析和应用

### 选项要求

- 每题4个选项（A、B、C、D）
- 只有1个正确答案
- 错误选项应该有迷惑性（似是而非）
- 避免使用"以上都对"或"以上都不对"

## 题目类型

### 1. 概念理解题

测试对核心概念定义和含义的理解。

**示例：**

```
技能的渐进式披露系统有几级？
   - A) 2级
   - B) 3级
   - C) 4级
   - D) 5级
```

**设计要点：**

- 直接询问概念的基本属性
- 选项应该是相关的数值或特征
- 错误选项应该看似合理

### 2. 原理应用题

测试对原理或方法的理解和应用能力。

**示例：**

```
以下哪个不是技能的核心设计原则？
   - A) 渐进式披露
   - B) 可组合性
   - C) 可移植性
   - D) 高性能
```

**设计要点：**

- 测试对原理分类的理解
- 选项应该是相关概念
- 考察辨析能力

### 3. 类比理解题

测试通过类比理解概念的能力。

**示例：**

```
在厨房类比中，MCP代表什么？
   - A) 食谱
   - B) 专业厨房
   - C) 厨师
   - D) 食材供应商
```

**设计要点：**

- 使用文档中的类比
- 选项应该都是类比中的元素
- 考察对类比的准确理解

### 4. 场景应用题

测试在具体场景中应用知识的能力。

**示例：**

```
定义技能用例时，建议定义几个具体的用例？
   - A) 1-2个
   - B) 2-3个
   - C) 5-10个
   - D) 越多越好
```

**设计要点：**

- 模拟实际使用场景
- 考察实践指导的理解
- 选项应该都是可行的选择

### 5. 流程理解题

测试对步骤或流程的理解。

**示例：**

```
渐进式披露的第一级是什么？
   - A) YAML前置元数据
   - B) SKILL.md主体
   - C) 链接文件
   - D) scripts文件夹
```

**设计要点：**

- 测试对顺序或层级的理解
- 选项应该都是流程中的元素
- 考察结构化知识

### 6. 对比辨析题

测试对相似概念的区分能力。

**示例：**

```
技能的可移植性意味着什么？
   - A) 只在Claude.ai中使用
   - B) 只在Claude Code中使用
   - C) 在所有Claude平台中工作方式相同
   - D) 需要根据平台修改
```

**设计要点：**

- 测试对概念差异的理解
- 选项应该反映不同的理解
- 考察精确性

### 7. 规则细节题

测试对具体规则或要求的记忆。

**示例：**

```
SKILL.md文件的命名要求是？
   - A) 不区分大小写
   - B) 必须是SKILL.md（区分大小写）
   - C) 可以是skill.md
   - D) 可以任意命名
```

**设计要点：**

- 测试对技术细节的掌握
- 选项应该包含常见的误解
- 考察精确性

### 8. 优势对比题

测试对优势或特点的理解。

**示例：**

```
以下哪项是技能的优势？
   - A) 需要每次重新解释偏好
   - B) 适用于一次性的独特任务
   - C) 不与Claude内置功能协作
   - D) 一次教导，处处受益
```

**设计要点：**

- 测试对优势的理解
- 包含一些看似合理但错误的选项
- 考察综合评价能力

## 错误选项设计技巧

### 1. 常见误解

选择学习者容易混淆或误解的内容作为错误选项。

**示例：**

```
技能文件夹命名应该使用什么格式？
   - A) 大写字母和空格
   - B) 下划线分隔（常见的错误方式）
   - C) kebab-case（小写字母和连字符）✓
   - D) 驼峰命名法
```

### 2. 部分正确

创建只包含部分正确信息的选项。

**示例：**

```
YAML前置元数据中description字段必须包含什么内容？
   - A) 技能的作者信息（不完全）
   - B) 技能做什么和何时使用它✓
   - C) 技能的许可证信息（不完全）
   - D) 技能的版本号（不完全）
```

### 3. 逻辑干扰

创建逻辑上看似合理但实际不相关的选项。

**示例：**

```
技能的主要指令内容位于哪个文件？
   - A) README.md（看似合理）
   - B) SKILL.md✓
   - C) metadata.yaml
   - D) config.md
```

### 4. 技术混淆

使用相似的技术术语创建混淆选项。

**示例：**

```
MCP +技能的故事应该强调什么？
   - A) 单独使用MCP的价值（部分正确）
   - B) 单独使用技能的价值（部分正确）
   - C) MCP和技能的协同价值✓
   - D) 技术实现细节（不相关）
```

## 质量检查清单

### 题目质量

- [ ] 题目表述清晰，无歧义
- [ ] 正确答案明确无误
- [ ] 错误选项看似合理但确实错误
- [ ] 避免使用暗示性语言
- [ ] 题干简洁明了

### 内容覆盖

- [ ] 涵盖本章所有核心概念
- [ ] 难度分布合理
- [ ] 包含不同类型的问题
- [ ] 题目数量适中（10-15题）

### 选项设计

- [ ] 选项长度基本一致
- [ ] 避免正确答案集中在某个位置（如总是C）
- [ ] 错误选项有迷惑性
- [ ] 不使用"以上都对"或"以上都不对"

## 避免的错误

### ❌ 常见问题

1. **题干过长**：导致阅读困难
2. **选项长度差异过大**：暗示正确答案
3. **答案规律明显**：如连续多个正确答案都是B
4. **错误选项太明显**：如包含明显错误信息
5. **题干表述含糊**：导致多个选项都可能正确
6. **使用否定题干**：如"以下哪个不是..."（慎用）

### ✅ 正确做法

1. 题干简洁，直指核心
2. 选项长度相近
3. 正确答案随机分布
4. 错误选项似是而非
5. 题干明确无歧义
6. 优先使用正面表述

## 出题流程

1. **回顾章节内容**：识别核心概念、关键要点
2. **确定题目类型**：为每个概念选择合适的题型
3. **设计题干**：清晰、简洁、直接
4. **编写选项**：确保1个正确、3个似是而非的错误
5. **检查质量**：对照质量检查清单
6. **调整平衡**：确保难度分布合理
7. **验证准确性**：确认答案无误
